# ADR-0008: プリセットの描画サムネイル保存

## コンテキスト

グラデーション設定だけの2Dサムネイルでは、Effect Stackによる最終的な見た目を一覧から判断できない。一方、各カードの表示時にWebGL描画を実行すると、カード数に応じてGPU負荷が増え、一覧のスクロールも不安定になる。

## 決定

- プリセット保存時に、低解像度の独立したWebGLコンテキストで1フレームを描画し、PNGデータURLをプリセットの任意フィールドとして保存する。
- 保存済みのPNGサムネイルはカード間で再利用し、カード表示時にWebGL描画を行わない。
- 旧プリセット、内蔵プリセット、描画に失敗したプリセットは、既存の軽量2Dフォールバック描画を使う。
- 外部画像入力はサムネイルに含めず、画像が必要な構成でも安全に描画できる入力なしの状態でキャプチャする。
- サムネイルは交換用JSON/ZIPへ含めるが、サイズ上限を設けて不正または過大な入力を拒否する。

## 理由

保存時の一度だけの描画でGPU負荷を限定し、一覧では静的画像を表示できる。任意フィールドにすることで既存の`Preset[]`とライブラリ形式を後方互換で読み込める。

## 結果

- 保存済みプリセットの一覧でEffect Stackを含む外観を確認できる。
- サムネイルデータによりライブラリの容量は増える。
- GPUやWebGLが利用できない環境でも保存と読み込みは継続できる。
