# ADR-0009: パラメータ制限を共有レジストリで管理する

## コンテキスト

同じ値に対するUI、ストア、プリセット、キーフレーム、WebGLの範囲処理が分散し、範囲外値や非有限値が描画境界まで到達する可能性がある。

## 決定

パラメータごとの既定値、範囲、step、整数性、角度単位を共有レジストリに集約し、各境界は同じnormalizerを呼び出す。GLSL側のclampはGPU破損時の最終防御として残す。

## 理由

設定経路が増えても、表示範囲と描画範囲の乖離をテストで検出でき、プリセット互換性も一箇所で管理できる。

## 代替案

| 案 | 採用しなかった理由 |
| --- | --- |
| UIだけでclampする | プリセット、キーフレーム、直接描画経路を保護できない。 |
| WebGLだけでclampする | 不正値がストアに残り、UIと保存データが不安定になる。 |

## 結果

### 利点

- 入力経路間で境界が一致する。
- 旧プリセットの補正と回帰テストを共通化できる。

### 欠点・コスト

- 新しい数値項目はレジストリ登録が必須になる。

## 再検討条件

パラメータが外部スキーマやGPU固有制限によって動的に変化し、静的レジストリで表現できなくなった場合。
