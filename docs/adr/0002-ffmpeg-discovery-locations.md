# ADR-0002: K-GG専用フォルダを優先してPATHからもFFmpegを検出する

## コンテキスト

Windowsデスクトップ版のMOV / MP4書き出しはPATH上の`ffmpeg`コマンドへ依存する。
PATH設定を不要にするK-GG専用配置を追加しつつ、既にPATHを設定済みの環境との互換性を
維持する必要がある。

実行ファイル横のフォルダはインストール先の書き込み権限とアプリ更新の影響を受ける。
一方、アプリローカルデータ領域は利用者権限で扱え、アプリ更新から分離できる。

## 決定

Windows x64 Tauri版の起動時に、アプリローカルデータ領域へ`ffmpeg`フォルダを作成する。
FFmpegは次の順序で探索・検証する。

1. `<app_local_data_dir>/ffmpeg/ffmpeg.exe`
2. システムPATH上の`ffmpeg`

専用フォルダの候補が有効なら常に優先する。存在しない、起動不能、または必要エンコーダーを
持たない場合はPATH候補へフォールバックする。
Windowsのパッケージ済みアプリが開発シェルと異なるPATHを継承する場合に備え、PATH候補は
プロセスの`PATH`に加えてHKCU/HKLMの環境変数`Path`を読み取り専用で参照して構成する。
K-GGはPATHまたはレジストリを変更しない。

K-GGはFFmpegをダウンロード、コピー、更新、削除しない。未検出時はgyan.devを外部ブラウザで
開く案内と専用フォルダを開く操作を提供し、利用者自身が配置またはPATH設定を行う。

K-GGはFFmpegをライブラリとしてリンクせず、コマンドラインとファイルだけを受け渡す
別プロセスとして実行する。書き出し直前にもRust側で候補を再確認する。

## 理由

- PATH設定なしでK-GG専用のFFmpegを利用できる。
- 既存のPATH利用者へ再配置を要求しない。
- アプリ更新やインストール先の書き込み権限から専用ファイルを分離できる。
- K-GGが第三者バイナリを配布・変更しない。
- 探索順序を固定し、実際に使用するFFmpegを利用者へ表示できる。

## 代替案

| 案 | 採用しなかった理由 |
| --- | --- |
| PATHだけを使用する | PATH設定なしの専用配置を提供できない |
| 実行ファイル横へ配置する | 更新と書き込み権限の影響を受ける |
| 専用フォルダだけを使用する | 既存のPATH環境との互換性を失う |
| 任意パスを設定として保存する | 永続設定と移動・削除時の回復処理が増える |
| 専用フォルダを優先してPATHへフォールバックする | 採用。専用利用と既存互換性を両立できる |

## 結果

### 利点

- 管理者権限なしで専用配置できる。
- PATH利用者の既存操作を維持できる。
- FFmpegの配布元やバージョンをK-GGが固定する必要がない。

### 欠点・コスト

- アプリデータ領域の場所をUIから開く機能が必要になる。
- 専用フォルダとPATHの両候補を検証し、優先順位と警告を管理する必要がある。
- Windowsではプロセス環境とレジストリ上の環境変数Pathの差分を考慮する必要がある。
- 利用者が配置したFFmpegのライセンス・安全性は利用者の選択に依存する。
- gyan.devの案内内容が変わる場合は文書更新が必要になる。
- Windows x64以外は別仕様が必要になる。

## 再検討条件

- Tauriのアプリデータ保存場所または更新方式が変わった。
- K-GGがFFmpegを直接ダウンロード、同梱、コピー、更新、削除する。
- 任意パス指定や複数FFmpegバージョン管理が必要になった。
- 動画エンコーダー要件または対象OSを変更する。
