# ADR-0003: 画像グラデーションのアンカー影響を永続設定として分離する

## コンテキスト

画像グラデーションは画像チャンネル値だけをGradient Rampの入力としていたため、グラデーションアンカーを動かしても画像の再配色をデザインできなかった。また、UV歪み後の座標で画像を再サンプルしていたため、Noiseなどが画像本体も変形していた。

## 決定

`ImageGradientConfig`へ0〜1の`anchorInfluence`を永続化する。画像サンプル用の`imageUV`は変形前の出力座標に固定し、変形済みの`gradientUV`はアンカー配色の計算だけに使用する。ランプ入力は画像チャンネル値とアンカー値を`anchorInfluence`で線形混合する。

新規設定と新規画像読込の既定値は0.5とする。`anchorInfluence`を持たない既存プリセットは0へ移行し、従来の見た目を維持する。

## 理由

- 画像の内容を保ったまま、アンカーと既存の歪みで配色を調整できる。
- 既存のアンカー型、キーフレーム、共通WebGL出力経路を再利用できる。
- 旧プリセットを意図せず再配色し直さない。

## 代替案

| 案 | 採用しなかった理由 |
| --- | --- |
| 画像のサンプルUVも歪ませ続ける | 画像本体が変形し、配色だけを調整したい用途に合わない。 |
| 歪みごとに画像・配色の対象を選択可能にする | 設定とUIが増え、今回の用途には過剰である。 |
| アンカー値を常に画像値へ加減算する | 高い強度で値が飽和し、アンカー配色を優先する操作ができない。 |

## 結果

### 利点

- Anchor Influenceで画像の明暗とアンカー配色の寄与を連続的に調整できる。
- UV歪みが画像の輪郭、Cover配置、入力アルファを変えない。

### 欠点・コスト

- 100%では画像の明暗ではなくアンカー配色が優先され、画像は主にアルファ形状として残る。
- プリセット読込時に旧設定の移行が必要になる。

## 再検討条件

歪みごとに画像本体へ適用する需要が確認された場合、個別の対象選択を別仕様として検討する。
