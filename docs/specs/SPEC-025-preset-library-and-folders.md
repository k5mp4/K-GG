# SPEC-025: プリセットライブラリとフォルダ階層

## 背景・問題

現在のプリセットUIは名前だけのフラットなリストで、内容を確認するために一つずつ読み込む必要がある。
また、ブラウザの`localStorage`とTauriの単一JSONファイルはどちらもプリセットの階層を表現できず、テーマ別の整理やフォルダ単位の共有ができない。

## ゴール・成功条件

- プリセットの内容を軽量な2Dサムネイルで一覧から判別できる。
- 保存済みプリセットをリストまたはグリッドで表示できる。
- 仮想フォルダを作成し、ネストした階層でプリセットを整理できる。
- フォルダカードに配下の代表5件を固定5分割で表示できる。
- 単一プリセット、現在フォルダ、ライブラリ全体を選択して書き出しできる。
- 旧形式の`Preset[]`と既存の保存データを失わずに読み込める。

## スコープ

### 対象

- `PresetLibrary`、`PresetFolder`、フォルダ所属と表示順を含むプリセットモデル。
- ブラウザ/Tauri共通のフォルダ操作とZIP形式のインポート/エクスポート。
- グラデーション、色、アンカー、グラデーション種別を描画する軽量プレビュー。
- フォルダツリー、フォルダカード、プリセットのリスト/グリッド表示。

### 対象外

- プレビューでのノイズ、ポストプロセス、外部画像の完全再現。
- OS上の実フォルダとの同期。
- 組み込みプリセットの編集、移動、ユーザーライブラリへの自動混在。
- フォルダ内データを破棄する削除操作。

## 方針

保存形式は、`version: 2`を持つライブラリ文書とする。ライブラリはフォルダ配列とプリセット配列を持ち、ルートは`folderId: null`で表す。フォルダ名は同じ親の中で一意とし、表示順は明示的な`order`で保持する。

既存のフラット配列はルート配下のライブラリへ正規化する。ブラウザの`localStorage`は正規化後に新形式で保存し、TauriのRustコマンドは旧配列と新オブジェクトの双方を読み込めるようにする。書き込みは既存のアトミック置換を維持する。

単一プリセットのJSONは従来形式を受け入れ、フォルダまたはライブラリ全体はライブラリ文書をZIPアーカイブへ格納する。インポートは選択中のフォルダへ追加し、IDは再採番する。既存データを置換する操作は提供しない。

プレビューは`GradientCanvas`の2Dフォールバック描画を共通関数へ抽出して使用する。フォルダカードでは配下を深さ優先・表示順で走査し、先頭5件を5分割表示する。5件未満の場合は空枠を表示する。

フォルダ削除時は、配下のプリセットと子フォルダを親へ移動する。名称変更・移動・保存先選択により階層を維持したまま整理できる。

## エラー・境界条件

- 空文字、パス区切り文字、制御文字を含むフォルダ名は拒否する。
- 同一親で重複するフォルダ名は拒否する。
- 破損したJSON/ZIP、未対応バージョン、空のアーカイブは既存ライブラリを変更せずエラー表示する。
- ZIPインポート時のID衝突は新しいIDへ変換する。
- 組み込みプリセットはルートの読み取り専用グループとして表示し、ユーザーライブラリの書き出しには含めない。

## 受け入れ条件

- AC-001: 旧形式の`Preset[]`を読み込むと、すべてのプリセットがルートフォルダ配下に表示される。
- AC-002: ユーザーがネストしたフォルダを作成し、そのフォルダを保存先に指定してプリセットを保存できる。
- AC-003: フォルダツリーで選択したフォルダの子フォルダとプリセットを、リスト/グリッドで切り替えて表示できる。
- AC-004: プリセットカードにグラデーションの軽量プレビューと名前が表示され、読み込み操作は既存のストア状態を復元する。
- AC-005: 子フォルダカードに配下の代表5件が固定5分割で表示され、件数が不足する場合は空枠になる。
- AC-006: 単一プリセット、現在フォルダ、ライブラリ全体を選択して書き出しでき、ZIPのインポート後も階層が維持される。
- AC-007: 不正なJSON/ZIPをインポートしても既存データが変更されず、利用者へエラーが表示される。
- AC-008: ブラウザの再読み込みとTauriの再起動後も、フォルダ階層とプリセットが復元される。
- AC-009: TypeScriptテスト、lint、build、ドキュメント検証が成功する。

## 移行・互換性

既存のブラウザ`localStorage`値とTauriの`presets.json`配列は読み込み時にルートフォルダへ移行する。既存JSONのインポートは継続して受け入れる。新形式を書き込んだ後も、旧アプリが新形式を読めることは保証しない。
