# SPEC-008: 画像カラーパレット生成機能

## 背景・問題

グラデーションを新規に作成する際、写真やイラストなどの画像からインスピレーションを得て色を抽出（カラーパレット化）し、それをグラデーションのキーカラーとして適用したいという強い需要があります。現状のK-GGでは、カラーコードを手動で一つずつ入力するか、組み込みのプリセットを使用するしかなく、直感的かつ素早い色のインポートができません。
また、抽出したカラーパレットを適用した後、それをプリセットとして保存するために別の「Preset」パネルへ移動して手動で再度名前を入力して保存する必要があり、抽出から保存にいたるまでのワークフローが分断されており非効率です。

## ゴール・成功条件

- ユーザーが用意した画像ファイル、または既存の「Image Overlay/Mask」に設定済みの画像から主要なカラー（2〜10色）を抽出できる。
- 抽出されたカラーパレットをワンクリックで現在のグラデーションストップ（等間隔）へ適用できる。
- 抽出されたカラーパレットに対して名前を指定し、グラデーションに適用された状態で直接プリセットとして新規保存（永続化）できる。
- 右サイドバーの最下部にコンパクトで美しい操作パネル（折りたたみ可能）が提供される。
- 画像解析およびクラスタリング処理が高速（50ms以下）に完了し、UIをフリーズさせない。

## スコープ

### 対象

- **パレット抽出アルゴリズムの実装**:
  - 外部ライブラリ（Color Thiefなど）に依存せず、軽量かつ保守性の高いMedian Cut（中央値カット）アルゴリズムをTypeScriptで自前実装する。
  - 解析の高速化のため、取得した画像を自動で一定サイズ（例: 128x128）以下にダウンサンプリングしてからピクセルデータを取得する。
- **入力ソースの選択**:
  - ローカルから直接画像を読み込む（ファイル選択およびドラッグ＆ドロップ対応）。
  - すでに読み込まれている「Image Overlay/Mask」の画像がある場合、ワンクリックでそれを流用できる。
- **抽出カスタマイズ**:
  - 抽出する色の数（2〜10色、デフォルト5色）を設定可能にする。
  - 抽出されたカラーパレットのソート機能（出現頻度順、輝度（Luminance）順、色相（Hue）順）の提供。これにより、グラデーションにしたときに滑らかなグラデーションを作りやすくする。
- **プリセット直接保存機能**:
  - パレット名のテキスト入力欄と「プリセットとして保存」ボタンを提供する。
  - 保存実行時に、現在のアプリ状態（その他のエフェクト・歪み設定を含む）のスナップショットをベースにし、グラデーション（stops）のみを抽出パレットに差し替えた状態でグラデーションプリセットとして保存する。
- **UIの実装**:
  - 右サイドバー（`DockPanel`）の最下部（`Bezier Distortion` の下）に「Color Palette Generator」セクションを追加。
  - 折りたたみ（`Collapsible`）可能にし、通常時は邪魔にならないようにする。
- **グラデーションへの適用**:
  - 抽出されたパレットを現在のグラデーションに適用するボタンを配置。適用時は等間隔でストップを配置する。

### 対象外

- カラーパレット単体のみの個別メタデータ保存（本機能では、グラデーションプリセットの仕組みをそのまま流用してグラデーションの一部として保存する）。
- 外部APIを利用した画像解析やカラー抽出（オフライン環境での動作とプライバシー保護のため、完全にフロントエンドのクライアントサイド処理とする）。

## 方針

### 1. アルゴリズム（Median Cut: 中央値カット法）
画像を128x128などの小さなサイズに描画するテンポラリCanvasを作成し、そのピクセルデータを `Uint8ClampedArray` で取得します。
その後、RGB空間においてMedian Cut法を実行して $K$ 個の代表色に分類します。
- **分割プロセス**: 
  1. すべての有効ピクセルを包含する最小のRGB直方体を計算します。
  2. 最も範囲（最大値 - 最小値）が広いRGBの軸を選択し、その軸上のピクセルの中央値（Median）を基準に直方体を2つの子ボックスに分割します。
  3. 分割後のボックス群のうち、最もピクセル数が多いボックス（またはRGB範囲が広いボックス）を選択し、目的の代表色数 $K$ に達するまでこの分割処理を再帰的に繰り返します。
  4. 分割された各ボックスに含まれるピクセル値の平均値を算出し、パレットの代表色とします。
- **高速化**: 128x128にダウンサンプリングすることで、処理するピクセル数を最大でも16,384個に抑え、瞬時（数ミリ秒）に処理を完了させます。
- **アルゴリズムの根拠と設計**:
  - 本機能で採用する中央値カット法は、Paul Heckbert氏のSIGGRAPH '82論文 [Color Image Quantization for Frame Buffer Display](https://dl.acm.org/doi/10.1145/965145.801294) で発表された古典的かつ実績のあるアルゴリズムを基礎としています。
  - また、画像の背景色など単一の広大な面積の色に支配されるのを防ぐため、Dan Bloomberg氏による [Modified Median Cut Quantization (MMCQ) / Leptonica](http://www.leptonica.org/color-quantization.html) の設計思想（JavaScriptの著名なライブラリ [color-thief](https://github.com/lokesh/color-thief) 等でも標準採用）を踏襲し、ピクセル数と体積の両面から分割優先度を動的に制御する実装アプローチを採ります。

### 2. ソートオプション
グラデーションは色の並びが重要であるため、以下のソート方法を提供します。
- **出現頻度順 (Dominance)**: 分割されたボックスに含まれるピクセル数が多い順（最も主要な色順）。
- **輝度順 (Luminance)**: 相対輝度 ($0.299R + 0.587G + 0.114B$) が小さい（暗い）順から大きい（明るい）順。
- **色相順 (Hue)**: HSLカラースペースに変換し、色相 $H$ の角度（0〜360度）が小さい順から大きい順。

### 3. グラデーションへの適用
抽出された色（Hex文字列）の配列 $C = [c_0, c_1, ..., c_{k-1}]$ に対して、グラデーションストップを次のように割り当てます。
$$position_i = \frac{i}{k-1}$$
例：5色の場合、`[0.0, 0.25, 0.5, 0.75, 1.0]` の位置にそれぞれの色を割り当て、現在のグラデーション `stops` を上書きします。

### 4. プリセットとしての保存
抽出したパレットを現在のグラデーション設定に反映した上で、`src/lib/presets` の `savePreset` 関数を呼び出し、現在のキャンバスサイズや歪み設定等の状態を含んだスナップショットをプリセットとして保存します。
キャンバスサイズ情報（`canvasW` / `canvasH`）は `App.tsx` から Props 経由で取得します。

## 代替案

| 案 | 採否 | 理由 |
| --- | --- | --- |
| `color-thief` 等の外部ライブラリの導入 | 不採用 | 追加のパッケージ依存によるビルドサイズ増加やセキュリティリスク、Tauri環境での互換性トラブルを避けるため。また、自前実装の方が軽量で柔軟にソートなどのカスタマイズができるため。 |
| K-Meansクラスタリングの採用 | 不採用 | 画像内の面積が大きな色（背景など）に強く引きずられやすく、面積の小さい特徴的なアクセントカラーが消滅しやすいため。グラデーションのキーカラー抽出にはMedian Cutのほうが適している。 |

## エラー・境界条件

- **画像データが取得できない、破損している**:
  - 画像読み込み失敗時には、パレット生成をスキップし、エラー文言「画像の読み込みに失敗しました」を表示する。
- **透明（アルファ）ピクセルの扱い**:
  - 透明度が一定以下（例: Alpha < 50）のピクセルはカラー抽出の対象から除外する（透明背景画像の場合に、無関係な黒や透明色がパレットに含まれるのを防ぐ）。
- **抽出色数 $K$ よりユニークな色数が少ない画像**:
  - 画像内のユニークな色数が $K$ 未満の場合は、見つかったユニークな色のみをパレットとし、残りのスロットは生成しない（空にする）。
- **保存時のプリセット名が空**:
  - プリセット名が入力されていない、またはスペースのみの場合は「プリセットとして保存」ボタンを非活性（Disabled）にし、誤保存を防ぐ。

## 受け入れ条件

- **AC-001**: 右サイドバーの最下部に「Color Palette Generator」という折りたたみ可能なセクションが表示されること。
- **AC-002**: セクション内に「画像を読み込む」ボタンがあり、画像を選択するとプレビュー画像および抽出されたパレット（デフォルト5色）が表示されること。また、ドラッグ＆ドロップでの画像読み込みに対応していること。
- **AC-003**: 「Image Overlayから取得」ボタンがあり、既存の「Image Overlay/Mask」で読み込まれている画像から直接パレットを抽出できること（画像が設定されていない場合は非活性または非表示）。
- **AC-004**: 抽出色数を2〜10色の間でスライダーまたは数値入力で変更でき、変更時にパレットが即座に再計算されて表示が更新されること。
- **AC-005**: パレットのソート方式を「出現頻度順」「輝度順」「色相順」からドロップダウン等で切り替えられ、切り替え時にプレビューパレットの順序が変化すること。
- **AC-006**: 「グラデーションに適用」ボタンを押すと、抽出された色が等間隔でグラデーションに反映され、Canvas側のグラデーション表示にリアルタイムで反映されること。
- **AC-007**: 大きな画像（例: 4K画像）を読み込ませても、ダウンサンプリング処理により、パレット生成処理がUIスレッドをフリーズさせずに50ms以内で高速に完了すること。
- **AC-008**: パレット抽出完了後、パレット名を入力するためのテキストボックスが表示されること。
- **AC-009**: パレット名が1文字以上入力されている場合のみ、「プリセットとして保存」ボタンが活性化されること。
- **AC-010**: 「プリセットとして保存」ボタンを押すと、抽出された色が等間隔でグラデーションに適用され、同時に指定した名前でプリセットとして新規保存（永続化）されること。また、PresetPanel等のプリセット一覧に即座に反映されること。

## 移行・互換性

プリセットの保存には既存の `savePreset` API を利用するため、保存データのフォーマット互換性に影響はありません。追加される入力欄の状態管理などは React のローカル State で行います。
